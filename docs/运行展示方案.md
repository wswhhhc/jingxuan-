# 菁选平台作品在线运行展示方案

## 1. 目标

让其他人上传的项目可以在当前菁选平台中运行展示，约束如下：

- 后端限定为 `Spring Boot`
- 前后端需要同时启动
- 当前版本先支持 `Windows 本机展示`
- 后续保留迁移到 `Linux + Docker` 的能力

当前版本的成功标准：

- 上传项目包后，平台能识别其是否为可运行项目
- 已发布作品在公开详情页可点击“在线体验”
- 点击后平台自动完成准备、分配资源、启动前后端
- 关闭页面后尽量停止，空闲 5 分钟自动回收
- 平台主业务数据与作品运行数据隔离

## 2. 总体设计

新增一个独立的 `runtime` 模块，负责以下能力：

- 解压项目包
- 扫描项目结构
- 调用 AI 生成标准运行描述文件 `jingxuan-demo.yml`
- 校验运行描述是否完整
- 分配前后端端口
- 为作品创建独立 MySQL 库
- 分配 Redis database index
- 启动和停止前后端进程
- 保存运行状态
- 处理心跳与超时回收

职责边界：

- AI：理解项目结构、生成 `jingxuan-demo.yml`、解释缺失项
- 平台代码：校验 manifest、启动/停止进程、分配资源、维护运行状态
- 前端页面：提供“在线体验”入口、预览页、心跳与关闭回收

## 4. 数据表设计

### 4.1 新增 `work_runtime`

在 `backend/sql/business/work_schema.sql` 中新增表：

```sql
CREATE TABLE IF NOT EXISTS work_runtime (
    id BIGINT NOT NULL COMMENT '运行记录ID',
    work_id BIGINT NOT NULL COMMENT '作品ID',
    status VARCHAR(20) NOT NULL COMMENT 'invalid/prepared/starting/running/failed/stopped',
    runtime_type VARCHAR(30) NOT NULL DEFAULT 'windows_process' COMMENT 'windows_process/docker',
    project_path VARCHAR(500) NOT NULL COMMENT '项目解压目录',
    manifest_path VARCHAR(500) DEFAULT NULL COMMENT 'manifest路径',
    backend_port INT DEFAULT NULL COMMENT '后端端口',
    frontend_port INT DEFAULT NULL COMMENT '前端端口',
    backend_pid BIGINT DEFAULT NULL COMMENT '后端进程PID',
    frontend_pid BIGINT DEFAULT NULL COMMENT '前端进程PID',
    preview_url VARCHAR(500) DEFAULT NULL COMMENT '平台预览地址',
    mysql_schema VARCHAR(100) DEFAULT NULL COMMENT '独立数据库名',
    redis_db INT DEFAULT NULL COMMENT 'Redis database',
    prepare_time DATETIME DEFAULT NULL COMMENT '准备完成时间',
    start_time DATETIME DEFAULT NULL COMMENT '启动时间',
    stop_time DATETIME DEFAULT NULL COMMENT '停止时间',
    last_access_time DATETIME DEFAULT NULL COMMENT '最后访问时间',
    error_message VARCHAR(1000) DEFAULT NULL COMMENT '错误信息',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted TINYINT NOT NULL DEFAULT 0,
    PRIMARY KEY (id),
    UNIQUE KEY uk_work_id (work_id),
    KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='作品运行状态表';
```

### 4.2 扩展 `port_manage`

新增 `port_type` 字段，区分：

- `backend`
- `frontend`

每个作品运行时必须同时占用两个端口。

### 4.3 运行数据隔离

- 平台主库：`jingxuan`
- 作品运行库：`jingxuan_demo_{workId}`
- Redis 主实例复用本机服务，作品使用独立 `database index`

## 5. Manifest 设计

平台自动生成固定文件：

- `jingxuan-demo.yml`

建议结构：

```yaml
version: 1

project:
  name: example-project
  type: springboot-web

backend:
  framework: springboot
  path: backend
  buildCommand:
    windows: mvn -DskipTests package
    linux: mvn -DskipTests package
  artifactPath: target/*.jar
  startCommand:
    windows: java -jar ${JAR_PATH}
    linux: java -jar ${JAR_PATH}
  healthPath: /actuator/health

frontend:
  path: frontend
  installCommand:
    windows: npm install
    linux: npm install
  startCommand:
    windows: npm run dev -- --host 127.0.0.1 --port ${FRONTEND_PORT}
    linux: npm run dev -- --host 0.0.0.0 --port ${FRONTEND_PORT}

database:
  mysql:
    enabled: true
    initSqlPath: sql/init.sql

cache:
  redis:
    enabled: true
```

平台强制覆盖的值：

- 数据库名：`jingxuan_demo_${WORK_ID}`
- 预览地址：`/preview/${WORK_ID}`
- 空闲超时：`5` 分钟
- 后端端口变量：`BACKEND_PORT`
- 前端端口变量：`FRONTEND_PORT`
- 前端 API 地址变量：`VITE_API_BASE_URL`

## 7. 接口设计

新增运行接口：

- `POST /api/runtime/{workId}/prepare`
- `POST /api/runtime/{workId}/start`
- `GET /api/runtime/{workId}/status`
- `POST /api/runtime/{workId}/heartbeat`
- `POST /api/runtime/{workId}/stop`

状态枚举固定为：

- `invalid`

- `running`
- `failed`
- `stopped`

返回字段统一包含：

- `message`

- `backendPort`
- `frontendPort`
- `missingFields`
- `missingFiles`
- `errorMessage`

## 8. 核心运行流程

### 8.1 `prepare`

1. 从 `work_attachment` 中找到项目 zip
2. 解压到 `uploads/projects/{workId}`
3. 扫描 `pom.xml`、`package.json`、`sql/init.sql`
4. 调用 AI 生成 `jingxuan-demo.yml`
5. 校验 manifest 和文件是否齐全
6. 持久化 `work_runtime.status=prepared` 或 `invalid`

### 8.2 `start`

1. 若未 `prepare`，先执行 `prepare`
2. 若已是 `running`，直接返回已有 `previewUrl`
3. 分配前后端端口
4. 创建独立库 `jingxuan_demo_{workId}`
5. 导入 `sql/init.sql`
6. 分配 Redis db
7. 启动 Spring Boot jar
8. 启动前端 `npm run dev`
9. 执行健康检查
10. 两者都成功后置为 `running`

任一步失败必须：

- 停掉已启动进程
- 释放端口
- 保留错误信息
- 状态置为 `failed`

### 8.3 `heartbeat`

- 仅更新 `last_access_time`

### 8.4 `stop`

- 停掉前后端 PID
- 释放双端口
- 保留运行记录
- 状态置为 `stopped`

### 8.5 超时回收

- 每分钟扫描一次 `running` 实例
- 若 `now - last_access_time > 5 分钟`，自动执行 `stop`

- `work_runtime` 表

- AI 生成 manifest

## 14. 测试重点

后端必测：

- 缺少 `pom.xml` 时不可运行
- 缺少 `package.json` 时不可运行
- 缺少 `sql/init.sql` 时返回明确缺失项
- `start` 失败时端口会释放
- 重复点击不会起两套实例

前端必测：

- 启动中状态可见
- 启动失败错误可见
- 预览页能轮询状态
- stop 后页面状态刷新正常

手工验收：

- 上传标准 Spring Boot 作品包
- 发布后点击“在线体验”
- 前后端同时运行
- 预览页能正常访问
- 关闭页面后 5 分钟内资源回收

### 14.1 样例联调包

仓库内提供了一份最小运行样例：

- `samples/runtime-demo-package`

结构：

- `backend/` Spring Boot 示例后端
- `frontend/` Vite + Vue 示例前端
- `sql/init.sql` 初始化 SQL

Windows PowerShell 可用以下命令打包：

```powershell
Compress-Archive -Path .\samples\runtime-demo-package\* -DestinationPath .\outputs\runtime-demo-package.zip -Force
```

建议联调步骤：

1. 将 `runtime-demo-package.zip` 作为作品附件上传
2. 创建或编辑作品并绑定该压缩包
3. 发布作品
4. 打开公开详情页并点击“在线体验”
5. 观察后台 `work_runtime` 状态、端口分配、前端预览页和后端 `/api/hello` 返回值

辅助脚本：

- `scripts/apply-runtime-support.ps1`
  - 将 `work_runtime` 表和 `port_manage.port_type` 应用到主库
- `scripts/start-jingxuan-backend.ps1`
  - 使用 `D:\java\jdk\bin\java.exe` 启动主平台后端
- `scripts/test-runtime-demo.ps1`
  - 登录、创建作品、上传样例包、调用 `prepare/start/status/stop`
- `scripts/run-runtime-validation.ps1`
  - 串联数据库准备、后端启动和样例联调

## 15. 后续演进

- Windows 本机

未来版本：

- Linux 云服务器

- 保持业务接口不变

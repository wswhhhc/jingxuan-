# 菁选平台作品在线运行展示方案

## 1. 当前结论

当前版本不再由菁选平台自动解压、构建、启动、停止学生上传的项目，也不再维护 `work_runtime`、`port_manage` 或 `/api/runtime/*` 接口。

在线体验采用更稳定、可演示、易维护的方式：

1. 学生上传作品资料、截图、视频、源码压缩包和运行说明。
2. 管理员审核通过后发布作品。
3. 管理员从已发布作品中选择少量优秀作品设置为精选。
4. 管理员维护该精选作品的 `previewUrl`，指向已经部署好的服务器访问地址。
5. 前台作品详情页检测到 `previewUrl` 后展示“在线体验”按钮，并在新窗口打开该地址。

该方案保留“精选作品可在线体验”的验收能力，同时把不同学生项目的 JDK、Node、数据库、端口、依赖安装和进程生命周期问题移出主平台，降低课程项目的实现和演示风险。

## 2. 适用范围

| 类型 | 当前处理方式 |
|------|--------------|
| 普通作品 | 展示标题、简介、技术栈、团队成员、封面、截图、视频、附件和运行说明 |
| 精选作品 | 在普通展示基础上增加 `previewUrl` 在线体验入口 |
| 已下线作品 | 前台不可见，在线体验入口不可见 |
| 未发布作品 | 仅管理员/相关角色可在后台或评分端查看，前台不可见 |

## 3. 数据落点

当前在线体验相关字段：

| 表 | 字段 | 说明 |
|----|------|------|
| `work` | `preview_url` | 服务器访问地址，当前前台详情页主要读取该字段 |
| `work_publish` | `featured` | 精选标记，`0=普通`、`1=精选` |
| `work_publish` | `preview_url` | 发布记录中的预览地址，历史兼容字段 |
| `work_publish` | `publish_status` | 发布状态，`0=未发布`、`1=已发布`、`2=已下线` |

旧库升级脚本：

```sql
backend/sql/business/2026-06-03-server-preview-clean-runtime.sql
```

该脚本会：

- 为 `work` 补充 `preview_url` 字段。
- 将 `work_publish.preview_url` 中已有地址回填到 `work.preview_url`。
- 删除已移除的 `work_runtime` 和 `port_manage` 表。

`backend/sql/business/2026-06-02-runtime-support.sql` 当前保留为空迁移，用于避免旧 runtime 表被重新创建。

## 4. 后端能力

当前在线体验由发布/精选能力承载，主要接口：

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/admin/audit/{workId}/publish` | 发布审核通过的作品 |
| POST | `/admin/audit/{workId}/offline` | 下线已发布作品 |
| POST | `/admin/audit/{workId}/featured` | 设置或取消精选，可携带 `previewUrl` |
| POST | `/api/publish/featured` | 发布模块精选设置接口 |
| GET | `/api/publish/{workId}` | 查询发布信息 |
| GET | `/public/works/{id}` | 前台作品详情，返回 `featured` 与 `previewUrl` |

实际控制器落点：

- `backend/src/main/java/com/jingxuan/modules/adapter/AdminApiController.java`
- `backend/src/main/java/com/jingxuan/modules/publish/controller/PublishController.java`
- `backend/src/main/java/com/jingxuan/modules/adapter/PublicApiController.java`

## 5. 前端能力

当前前端落点：

- 管理端审核页：`frontend/src/views/admin/audit/index.vue`
  - 已通过作品可发布。
  - 已发布作品可下线。
  - 已发布作品可设置或取消精选。
  - 详情中展示服务器地址。
- 前台作品详情页：`frontend/src/views/public/WorkDetail.vue`
  - 当作品存在 `previewUrl` 时展示“在线体验”按钮。
  - 点击后调用 `window.open(normalizePreviewUrl(previewUrl), '_blank')`。
  - 相对地址会按当前站点补齐，完整 URL 直接打开。

当前没有以下前端文件：

- `frontend/src/views/public/WorkPreview.vue`
- `frontend/src/views/admin/port/index.vue`
- `frontend/src/api/runtime.ts`

## 6. 推荐部署方式

### 6.1 直接填写完整 URL

适用于学生项目已部署在独立域名或端口上的场景。

示例：

```text
https://demo.example.edu/jingxuan-work-1001/
http://192.168.1.20:9003/
```

管理员将该地址填入精选作品的预览地址后，前台详情页会直接打开。

### 6.2 使用 Nginx 统一入口

适用于希望所有体验地址都挂在主平台域名下的场景。

示例：

```nginx
location /preview/1001/ {
    proxy_pass http://127.0.0.1:9003/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

然后在平台中填写：

```text
/preview/1001/
```

仓库内的 `docs/nginx-preview.conf` 可作为反向代理参考。

## 7. 管理员操作流程

1. 学生提交作品并上传必要附件。
2. 管理员在 `/admin/audit` 审核通过作品。
3. 管理员点击“发布展示”。
4. 管理员在已发布作品上点击“设为精选”。
5. 如有可访问的部署地址，维护 `previewUrl`。
6. 游客打开 `/works/{id}`，点击“在线体验”。

## 8. 验收标准

| 验收项 | 标准 |
|--------|------|
| 已发布作品展示 | 前台作品列表和详情页可访问 |
| 未发布/已下线隔离 | 前台不可见 |
| 精选标记 | 管理端可设置/取消，前台显示精选标识 |
| 在线体验入口 | 存在 `previewUrl` 时展示按钮，不存在时不展示 |
| 地址打开 | 完整 URL 或相对 `/preview/{id}` 地址可正常新窗口打开 |
| 反向代理 | 使用 Nginx 时 `/preview/{id}` 能访问目标学生项目 |

## 9. 旧 runtime 方案处理

仓库中仍保留了一些历史辅助文件和样例包，主要用于说明和回溯，不代表当前平台仍支持自动运行学生项目：

- `samples/runtime-demo-package/`
- `outputs/runtime-demo-package.zip`
- `scripts/apply-runtime-support.ps1`
- `scripts/test-runtime-demo.ps1`
- `scripts/run-runtime-validation.ps1`

不要把这些脚本作为当前验收流程使用。若后续要重新引入自动运行能力，应重新设计 runtime 模块，并明确隔离进程、数据库、端口、资源回收、日志、安全沙箱和 Linux/Docker 部署方案。

## 10. 后续可选演进

如项目后续需要恢复“上传后自动运行”能力，建议作为独立二期处理：

- 新增 `modules/runtime`，而不是混入发布/精选流程。
- 使用容器隔离学生项目，而不是直接在主机启动任意脚本。
- 每个项目使用独立数据库、网络、文件目录和日志目录。
- 后端提供 `prepare/start/status/stop/heartbeat` 接口。
- 前端新增预览承载页和管理员运行管理页。
- 对上传包做白名单、资源限额、超时回收和审计。

在这些条件具备前，当前 `previewUrl` 方案是推荐实现。

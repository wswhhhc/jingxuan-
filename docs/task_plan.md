# 任务计划：学院作品展示平台

## 目标
完成学院作品展示平台前后端全栈开发，实现作品提交、审核、评分、排行榜、精选展示的完整闭环。

## 分工角色
- `组长`：后端 1，负责架构、认证、审核、部署类任务。
- `后端 2`：负责作品、评分、排行、通知类业务。
- `前端 1`：负责学生端、提交页、作品浏览页。
- `前端 2`：负责教师端、后台端、管理页。

## 执行原则
- 按 `P0 → P1 → P2` 推进。
- 每张任务卡都要有负责人、交付物和完成标准。
- 先打通主链路，再补增强功能。
- 先定接口、状态和数据结构，再并行开发前后端。

阶段 14：最终收尾回归验证 + 文档同步（2026-05-19）

## 新增：联调阻塞问题

当前发现了一个阻塞联调的问题，需要作为阶段 8.2 的前置条件优先修复：

| 问题 | 发现时间 | 涉及模块 | 描述 | 影响 | 状态 |
|------|----------|----------|------|------|------|
| 登录后页面未跳转 | 2026-05-18 | 前端 1 / router | 登录 API 200 成功，token 已下发，但页面卡在 /login，未进入 /student/home | 阻塞所有角色进入系统，优先级 P0 | ✅ 已修复 |

**根因（3 个 Bug 叠加）：**
1. router 守卫只读 `localStorage`，默认登录存 `sessionStorage` → 守卫每次拦截回登录页
2. store token 初始化也只读 `localStorage` → 刷新后 token 丢失
3. router 守卫检查 `to.meta.roles` 但子路由不继承父路由 meta，所有受保护页面对守卫不可见

**修复后验证：** 需要学生/教师/管理员三种角色登录验证

**修复负责人：** 前端 1 ✅ 已修复
**修复后需验证：** 学生/教师/管理员 三种角色均能登录并进入各自首页

**修改文件：**
- `frontend/src/router/index.ts` — 守卫检查双存储 + 遍历匹配路由链
- `frontend/src/stores/student/auth.ts` — token 初始化读双存储 + login 写双存储
- `frontend/src/views/student/Login.vue` — 角色感知跳转 + 错误提示

## 阶段 15：教师端体验补强规划（2026-05-19）
目标：把教师端从“可评分”升级为“可高效工作”，补齐入口、回看、进度和评分依据展示，减少教师在评分过程中的来回切换和信息缺口。

### 现状判断

当前教师端已有：
- `作品评分`
- `排行榜`
- `消息通知`

当前教师端缺少：
- 教师首页/工作台
- 评分历史回看入口
- 评分进度统计
- 只看未评分筛选
- 作品预览、附件、演示材料的集中查看入口

### 规划原则

- 先补“评分闭环”相关功能，再补“管理感知”功能。
- 优先复用现有后端接口，减少无必要的新接口开发。
- 教师端继续保持匿名评审边界，不暴露学生身份。
- 每一阶段都要有可独立上线的用户价值。

### 需求分级

## 关键问题
1. DeepSeek 内容审核是否先做连通性验证和兜底策略？
2. 文件存储是否保持本地优先，MinIO 仅作为可选增强？
3. 端到端演示数据是否从建表阶段开始同步准备？

## 已做决策
| 决策 | 理由 |
|------|------|
| 前后端分离架构 | Vue 3 + Spring Boot，便于 4 人分工 |
| JWT 无状态认证 | 适合前后端分离，无需服务端 session |
| LLM 内容审核用 DeepSeek 官方 API | 用户指定，模型 deepseek-v4-flash |
| 数据库优先建表再写代码 | 需求文档已有完整 19 表设计 |
| P0→P1→P2 递进开发 | 确保核心闭环先跑通 |
| 本地文件存储优先 | 更适合实训环境，MinIO 作为可选项 |

## 3. 已知问题

### 3.1 测试层面的已知问题

| 编号 | 问题 | 影响范围 | 原因 |
|------|------|---------|------|
| K1 | `WorkServiceImpl.shouldThrowWhenMemberAlreadyInBatch` 被跳过 | 1 个单元测试 | Mockito + MyBatis-Plus `LambdaQueryWrapper` 序列化兼容问题，需在集成测试中覆盖 |
| K2 | `EndToEndFlowTest.scoreLifecycle` 评分 500 错误 | 1 个集成测试 | 评分插入时雪花 ID 为空，可能与事务回滚后自增 ID 重置有关 |
| K3 | `PublicApiTest.shouldReturn429WhenRateLimited` 限流测试 | 1 个集成测试 | 并发请求时序问题，偶发性失败 |

### 3.2 产品层面的已知缺陷

| 编号 | 缺陷 | 严重度 | 状态 |
|------|------|--------|------|
| B1 | 精选作品弹窗缺少「预览地址」输入框（已修复） | P2 | ✅ 已修复 |
| B2 | 无 `previewUrl` 的作品提交审核时返回 400 不够明确 | P3 | 待优化提示文案 |
| B3 | 删除已发布作品前必须先下线，前端无明确引导 | P3 | 体验优化 |

### 3.3 环境依赖

- 集成测试需要 MySQL 8.0 和 Redis 7 运行环境
- DeepSeek API 在测试环境中 fallback 为 `bypass`，不影响流程
- 文件上传测试需确保 `uploads/` 目录存在且有写入权限

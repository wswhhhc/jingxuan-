## 5. 缺陷统计

### 5.1 测试过程发现/修复的缺陷

| 编号 | 缺陷 | 级别 | 发现阶段 | 状态 |
|------|------|------|---------|------|
| B1 | 精选弹窗缺少「预览地址」输入框 | P2 | 自动化测试 | ✅ 已修复 |
| B2 | 提交审核时 previewUrl 为空导致不明 400 | P3 | 集成测试 | ✅ 已修复 |
| B3 | Mockito+MyBatis-Plus LambdaQueryWrapper 兼容 | P3 | 单元测试 | ✅ 已修复（改用 QueryWrapper） |
| B4 | 评分插入时雪花 ID 为空导致 500 | P2 | E2E 测试 | ✅ 已修复（预生成 ID） |
| B5 | init_schema.sql 硬编码 USE jingxuan 导致测试库无表 | P2 | 环境准备 | ✅ 已修复（测试库使用 sed 跳转） |

### 5.2 已知遗留问题

| 编号 | 问题 | 级别 | 原因 | 建议 |
|------|------|------|------|------|
| R1 | FileUploadTest 超大压缩包（~200MB）上传返回 200 而非 400 | P3 | 测试环境配置的多部分文件上限 >210MB | 对齐生产环境限制 |
| R2 | 删除已发布作品前必须先下线，前端无明确引导 | P3 | 体验设计不足 | 优化提示文案 |

## 8. 风险评估与建议

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| DeepSeek API 在测试环境不可用 | 内容安全审核跳过 | 生产环境需配置 API Key |
| 集成测试依赖 MySQL+Redis | 无法在无数据库环境运行 | 已创建 jingxuan_test 数据库并导入全部迁移 |
| 文件上传需磁盘空间 | 超大文件上传测试可能失败 | 准备 2GB+ 空闲空间 |

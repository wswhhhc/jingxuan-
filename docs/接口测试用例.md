## 阅读指南

本文档按角色分为 5 个章节，每个 API 包含正向用例、异常用例和边界用例。

**返回约定：**
- 成功返回 `Result<T>`：`{ "code": 200, "msg": "success", "data": T }`
- 业务异常返回 `Result.fail(msg)`：`{ "code": 400, "msg": "...", "data": null }`
- 未认证返回 401，无权限返回 403，资源不存在返回 404

**通用前置条件：**
- 认证类 API 需在 `Authorization` 头中携带 `Bearer <token>`
- 测试前需执行 `test_data.sql` 初始化测试数据
- 角色权限：ADMIN=3, TEACHER=2, STUDENT=1

**覆盖标记：**
- `[自动]` — 已被自动化集成测试覆盖
- `[手动]` — 仅适合手动测试或需要特殊环境
- `[未覆盖]` — 尚未覆盖

### 4.3 POST `/teacher/score` — 提交评分 `[自动]`

**请求体：**
```json
{"workId":1,"innovation":20,"difficulty":20,"completion":25,"practicality":15,"comment":"优秀"}
```

**正向用例：**

| 用例 | 前置 | 预期 |
|------|------|------|
| 首次评分 | 作品已通过+批次进行中 | code=200, 插入新评分记录 |
| 更新评分 | 已评过分 | code=200, 更新已有记录 |

**异常用例：**

| 用例 | 前置 | 预期 |
|------|------|------|
| 作品不存在 | workId=999 | code=400 |
| 作品未通过审核 | status≠3 | code=400 |
| 评分超出范围 | innovation=30(>25) | 参数校验失败 400 |
| 批次未开始/已结束 | batch 时间范围外 | code=400 |
| 评语含违规内容 | comment="辱骂内容" | code=400（非 bypass 模式） |
| DeepSeek 不可用但 fallback=bypass | 审核服务异常 | 评分仍可成功提交 |
| DeepSeek 不可用且 fallback=reject | 审核服务异常 | code=400 |

**边界用例：**

| 字段 | 最小值 | 最大值 |
|------|--------|--------|
| innovation | 0 | 25 |
| difficulty | 0 | 25 |
| completion | 0 | 30 |
| practicality | 0 | 20 |
| 总分范围 | 0 | 100 |

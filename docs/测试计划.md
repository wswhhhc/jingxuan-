## 1. 测试目标

验证菁选校园作品展示平台的核心功能是否满足需求，确保学生、教师、管理员、游客四端之间的业务流程完整闭环，覆盖正常流程与异常场景。

### 测试范围

- 学生端：作品 CRUD、提交审核、查看评分排行、消息通知
- 教师端：四维度评分、历史查询、排行榜查看/刷新、消息通知
- 管理端：审核通过/驳回、发布/下线/精选、评分批次管理、用户/角色/权限管理、字典/公告/通知/日志
- 前台公开端：作品浏览、详情、评论、点赞、排行榜浏览
- 跨端流程：提交→审核→评分→排行→发布 全链路

### 3.2 手工测试（核心业务流）

#### 测试流 1：学生提交 → 审核 → 发布（P0）

```
前置条件：学生 2022001 已登录

1. 学生进入「提交作品」页面
2. 填写作品信息（名称、简介、技术栈、成员）
3. 上传附件和演示视频
4. 点击「提交审核」
   → 预期：状态变为「已提交」
5. 切换管理员账号 admin 登录
6. 进入「审核管理」列表
7. 找到刚提交的作品
8. 点击「审核通过」
   → 预期：状态变为「已通过」
9. 点击「发布展示」
   → 预期：前台可见该作品
10. 切换游客身份访问首页
    → 预期：作品出现在列表中
```

#### 测试流 2：教师评分 → 排行榜（P0）

```
前置条件：至少 3 个已通过作品

1. 教师 t001 登录
2. 进入「作品评分」页面
3. 对作品 A 打分：创新 20 / 难度 18 / 完成 25 / 实用 15
4. 提交评分
   → 预期：评分成功，不可重复提交
5. 教师 t002 登录
6. 对同一作品 A 打分
7. 管理员进入「评审批次管理」
8. 点击「公示排行榜」
   → 预期：前台排行榜展示作品 A、B、C 的平均分排序
9. 切换游客身份访问排行榜
   → 预期：排名正确，显示平均分与评分人数
```

#### 测试流 3：异常场景（P1）

| 场景 | 操作 | 预期 |
|------|------|------|
| 重复提交 | 审核通过后再次提交 | 提示当前状态不允许操作 |
| 空标题创建 | 不填作品名称直接提交 | 表单校验拦截 |
| 无视频提交 | 不上传演示视频直接提交 | 提示"请上传演示视频" |
| 驳回后重新提交 | 管理员驳回 → 学生修改 → 重新提交 | 状态正常流转 |
| 越权访问 | 学生访问 `/admin/audit` | 重定向或 403 |
| 删除已发布作品 | 管理员下线 → 学生删除 | 必须先下线才能删除 |
| 敏感词评论 | 评论包含敏感词 | 评论被拦截，提示违规 |

## 8. 缺陷管理

缺陷按严重程度分级：

| 级别 | 定义 | 响应时间 |
|------|------|---------|
| **P0 致命** | 核心流程阻塞、数据丢失、安全漏洞 | 立即修复 |
| **P1 严重** | 功能不可用、错误数据展示 | 24h 内修复 |
| **P2 一般** | 非核心功能异常、UI 问题 | 3 天内修复 |
| **P3 建议** | 样式微调、体验优化 | 迭代规划 |

缺陷报告模板：

```
标题: [P0] 学生提交作品后管理员审核列表不显示
环境: 测试环境
前置: 学生 2022001 已提交作品
步骤: 1. admin 登录 → 2. 进入审核管理
结果: 列表为空，F12 显示接口 500
预期: 显示待审核作品列表
附件: (截图/日志)
```

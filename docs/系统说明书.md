# 学院作品展示平台 — 系统说明书

## 1. 项目概述

学院作品展示平台是面向 IT 学院内部的学生作品集中管理、展示、评分与排行系统。
主要服务于课程作品、实训项目和毕业设计的提交、审核、评分、展示与精选发布的全流程管理。
平台覆盖学生、教师、管理员三种角色，实现从作品上传到精选展示的完整闭环。

## 3. 部署指南

### 3.1 环境要求

- JDK 17+
- Node.js 20+
- MySQL 8.0
- Redis 7

### 3.2 数据库初始化

按顺序执行以下 SQL 脚本：

```bash
# 1. 基础结构（用户、角色、菜单、字典、日志、公告、通知）
mysql -u root -p < backend/sql/base/init_schema.sql

# 2. 业务表结构（作品、成员、附件、审核、发布、评分、评论、批次、规则、奖项）
mysql -u root -p < backend/sql/business/work_schema.sql

# 3. 测试数据（学生/教师/作品/评分/公告）
mysql -u root -p < backend/sql/business/test_data.sql
```

旧库升级时，继续按日期执行 `backend/sql/business/yyyy-MM-dd-*.sql` 增量脚本。当前版本的 `2026-06-03-server-preview-clean-runtime.sql` 会把在线体验地址收敛到 `work.preview_url`，并清理已移除的本机 runtime/port 表。

### 3.3 后端启动

```bash
cd backend
mvn spring-boot:run
```

或使用 JAR 包启动：

```bash
cd backend
mvn package -DskipTests
java -jar target/jingxuan-backend-1.0.0.jar --spring.profiles.active=dev
```

后端默认运行在 http://localhost:8080

接口文档（Knife4j）：http://localhost:8080/doc.html

### 3.4 前端启动

```bash
cd frontend
npm install
npm run dev
```

前端默认运行在 http://localhost:5173

### 3.5 配置说明

核心配置位于 `backend/src/main/resources/application.yml`：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `spring.datasource.url` | 数据库连接地址 | `jdbc:mysql://localhost:3306/jingxuan` |
| `spring.datasource.username` | 数据库用户名 | `root` |
| `spring.datasource.password` | 数据库密码 | 环境变量 `DB_PASSWORD` |
| `jwt.secret` | JWT 签名密钥 | 内置 |
| `jwt.expiration` | Token 过期时间 | 86400000ms (24h) |
| `deepseek.api-key` | DeepSeek API 密钥 | 环境变量 `DEEPSEEK_API_KEY` |
| `deepseek.fallback` | 内容审核失败策略 | dev=`bypass`，默认=`reject` |
| `jingxuan.upload.path` | 文件上传存储路径 | `./uploads` |

## 6. 核心业务流程

```
学生提交作品
    ↓
管理员审核（通过/驳回）
    ↓ 通过
教师匿名评分（创新性/技术难度/完成度/实用性）
    ↓
排行榜生成（自动/手动刷新）
    ↓
管理员发布作品（前台可见）
    ↓
管理员设置精选与服务器访问地址
```

### 审核状态流转

```
草稿(DRAFT, 0) → 已提交(SUBMITTED, 1) → 已通过(APPROVED, 3)
                   ↓                        ↓
                已驳回(REJECTED, 2)       发布/下线
```

### 发布状态流转

```
未发布(UNPUBLISHED, 0) → 已发布(PUBLISHED, 1) → 已下线(OFFLINE, 2)
```

### 评分规则

| 维度 | 满分 | 说明 |
|------|------|------|
| 创新性 | 25 | 作品创意与创新程度 |
| 技术难度 | 25 | 技术实现复杂度 |
| 完成度 | 30 | 功能完整性与代码质量 |
| 实用性 | 20 | 实际应用价值 |

- 总分 100 分，每位教师对同一作品只能提交 1 次正式评分（截止前可修改）
- 多位教师评分时取平均分，保留两位小数
- 排名相同按"完成度 → 创新性 → 提交时间"排序

## 10. 项目结构

### 10.1 后端结构

```
backend/
├── src/main/java/com/jingxuan/
│   ├── auth/             # 登录、用户信息、改密、登出
│   ├── common/           # Result、PageResult、BaseEntity、分页工具
│   ├── config/           # 安全、Jackson、Knife4j、MyBatis-Plus、DeepSeek、WebMvc
│   ├── controller/       # 用户、角色、菜单管理
│   ├── entity/           # 实体类
│   ├── mapper/           # MyBatis-Plus Mapper 接口
│   ├── modules/          # 业务模块与四端适配 API
│   │   ├── adapter/      # Admin/Teacher/Student/Public API
│   │   ├── work/         # 作品、附件、成员
```

## 12. 常见问题排查

### 端口占用
如果启动时提示端口被占用，请运行项目根目录的 `stop.bat` 关闭相关进程，或手动检查：
- 后端 8080 端口：`netstat -ano | findstr 8080`
- 前端 5173 端口：`netstat -ano | findstr 5173`

### 数据库连接失败
- 确认 MySQL 服务已启动：`net start mysql` 或通过服务管理器启动
- 检查 `application.yml` 中的数据库连接地址、用户名、密码是否正确
- 确认 jingxuan 数据库已创建，SQL 脚本已完整执行

### DeepSeek API 不可用
- 开发环境（dev profile）下，DeepSeek API 调用失败会自动 bypass，不影响审核功能正常使用
- 生产环境需配置 `DEEPSEEK_API_KEY` 环境变量
- 可在管理后台「审核规则」页面进行连通性测试

### 前端页面白屏 / 接口 404
- 确认后端已启动并可访问：`curl http://localhost:8080/doc.html`
- 确认前端代理配置正确：`frontend/vite.config.ts` 中 `/api` 代理指向 `http://localhost:8080`
- 浏览器 F12 → Network 面板查看具体请求错误

### 文件上传失败
- 检查 `jingxuan.upload.path` 配置的目录是否存在且有写入权限
- 默认上传路径为项目根目录下的 `./uploads`
- 确认 `application.yml` 中 multipart 大小限制配置满足需求（当前单文件 1536MB、单次请求 1600MB）

# 冒烟测试

| 文档信息 | 内容 |
|---------|------|
| 项目名称 | 菁选校园作品展示平台 |
| 测试类型 | 冒烟测试（Smoke Test） |
| 执行时机 | 每次部署后 / PM2 重启后 |
| 预估耗时 | 3~5 分钟 |

---

## 1. 什么是冒烟测试

冒烟测试在每次部署后执行，快速验证系统核心功能是否正常。**不追求全面覆盖，只确认关键路径可用。** 全部通过则系统可以上线；失败则退回修复。

---

## 2. 快速验证脚本

将以下脚本保存为 `smoke-test.sh`，部署后执行：

```bash
#!/usr/bin/env bash
# 菁选校园作品展示平台 — 冒烟测试脚本
set -euo pipefail

BASE_URL="${1:-http://localhost:8080}"
PASS=0
FAIL=0

check() {
    local desc="$1" method="$2" url="$3" expect="$4" token="$5"
    local extra=()
    [ -n "$token" ] && extra=(-H "Authorization: Bearer $token")
    
    if [ "$method" = "POST" ]; then
        local data="${6:-}"
        local resp
        resp=$(curl -s -o /tmp/smoke-resp.txt -w "%{http_code}" -X POST \
            "${extra[@]}" -H "Content-Type: application/json" \
            -d "$data" "$BASE_URL$url" 2>/dev/null || echo "000")
    else
        local resp
        resp=$(curl -s -o /tmp/smoke-resp.txt -w "%{http_code}" \
            "${extra[@]}" "$BASE_URL$url" 2>/dev/null || echo "000")
    fi
    
    if echo "$resp" | grep -qE "$expect"; then
        echo "  ✅ $desc"
        PASS=$((PASS + 1))
    else
        echo "  ❌ $desc (期望 $expect, 实际 $resp)"
        cat /tmp/smoke-resp.txt | head -c 200
        echo ""
        FAIL=$((FAIL + 1))
    fi
}

echo ""
echo "========================================"
echo "  菁选 — 冒烟测试"
echo "  目标: $BASE_URL"
echo "========================================"
echo ""

# 获取 token
ADMIN_TOKEN=$(curl -s "$BASE_URL/auth/login" -X POST \
    -H "Content-Type: application/json" \
    -d '{"username":"admin","password":"admin123"}' | \
    python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])" 2>/dev/null || echo "")
STU_TOKEN=$(curl -s "$BASE_URL/auth/login" -X POST \
    -H "Content-Type: application/json" \
    -d '{"username":"2022001","password":"test123"}' | \
    python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])" 2>/dev/null || echo "")

# 1. 认证
echo "【1/6】认证"
check "后端启动"              GET  "/auth/login"            "200|405" "" ""
check "管理端登录"           POST "/auth/login"            "200" ""     '{"username":"admin","password":"admin123"}'
check "学生登录"             POST "/auth/login"            "200" ""     '{"username":"2022001","password":"test123"}'
check "错误密码拒绝"         POST "/auth/login"            "401" ""     '{"username":"admin","password":"wrong"}'

# 2. 管理端
echo "【2/6】管理端"
check "仪表盘统计"           GET  "/admin/dashboard/stats" "200" "$ADMIN_TOKEN"
check "审核列表"             GET  "/admin/audit/list"      "200" "$ADMIN_TOKEN"
check "用户列表"             GET  "/admin/users"           "200" "$ADMIN_TOKEN"

# 3. 教师端
echo "【3/6】教师端"
TEA_TOKEN=$(curl -s "$BASE_URL/auth/login" -X POST \
    -H "Content-Type: application/json" \
    -d '{"username":"t001","password":"test123"}' | \
    python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])" 2>/dev/null || echo "")
check "待评分作品列表"       GET  "/teacher/work/list"      "200" "$TEA_TOKEN"
check "教师排行榜"           GET  "/teacher/ranking/list"   "200" "$TEA_TOKEN"

# 4. 学生端
echo "【4/6】学生端"
check "我的作品列表"         GET  "/student/works"          "200" "$STU_TOKEN"
check "创建作品"             POST "/student/works"          "200" "$STU_TOKEN" \
    '{"title":"冒烟测试作品","summary":"冒烟测试","techStack":"Java"}'

# 5. 前台
echo "【5/6】前台"
check "公开作品列表"         GET  "/public/works"           "200" ""
check "公开排行榜"           GET  "/public/ranking/list"    "200" ""

# 6. 安全
echo "【6/6】安全"
check "越权拦截（学生→教师）" GET  "/teacher/work/list"    "403" "$STU_TOKEN"
check "匿名访问拦截"         GET  "/student/works"          "401" ""

echo ""
echo "========================================"
echo "  结果: $PASS 通过 / $FAIL 失败"
echo "========================================"
[ "$FAIL" -eq 0 ] && echo "  ✅ 冒烟测试通过" || echo "  ❌ 存在失败项"
echo ""
```

### 执行方式

```bash
# 保存脚本
chmod +x /opt/jingxuan/scripts/smoke-test.sh

# 对本地后端执行
bash /opt/jingxuan/scripts/smoke-test.sh http://localhost:8080

# 对远程服务器执行
bash /opt/jingxuan/scripts/smoke-test.sh http://your-server-ip
```

## 3. 快速手工检查（无脚本环境）

如果服务器没有 curl/python3，可用以下最小检查清单：

| # | 检查项 | 方法 | 预期 |
|---|--------|------|------|
| 1 | 后端进程运行 | `pm2 status` | jingxuan-back 状态 online |
| 2 | 登录页可访问 | 浏览器打开 `http://ip/` | 页面正常加载（非白屏/502） |
| 3 | 管理端登录 | 浏览器登录 admin/admin123 | 跳转控制台 |
| 4 | 前台作品列表 | 浏览器访问 `/works` | 显示作品 |
| 5 | 后端日志无异常 | `pm2 logs jingxuan-back --lines 20` | 无连续报错 |

---

## 4. 通过标准

冒烟测试通过条件：**全部 12 项检查均通过**。

| 项目 | 标准 |
|------|------|
| 认证 | 后端可响应 / 管理员/学生正常登录 / 错误密码被拒 |
| 管理端 | 仪表盘 / 审核列表 / 用户列表 正常返回 |
| 教师端 | 待评分作品列表 / 排行榜 正常返回 |
| 学生端 | 我的作品 / 创建作品 正常返回 |
| 前台 | 作品列表 / 排行榜 正常返回 |
| 安全 | 越权 / 匿名访问 正确拦截 |

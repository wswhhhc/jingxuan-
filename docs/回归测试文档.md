## 1. 回归测试策略

### 1.1 触发条件

以下情况必须执行回归测试：

| 触发场景 | 说明 | 回归范围 |
|---------|------|---------|
| 核心代码变更 | WorkService / ScoreService / AuditService 等核心 Service 修改 | 全部 P0 回归 |
| 安全修复 | JWT / 权限 / 鉴权相关修改 | 认证 + 权限测试 |
| 数据库 Schema 变更 | 新增/修改表结构或字段 | 所有受影响的 API |
| 依赖升级 | Spring Boot / MyBatis-Plus / 其他关键依赖版本变更 | 全部回归 |
| 发布前 | 每次正式发布前 | 全部回归（含端到端） |

### 1.2 回归测试等级

| 等级 | 范围 | 执行时机 | 预估耗时 |
|------|------|---------|---------|
| P0 快速回归 | 认证 + 作品 CRUD + 评分 + 审核 + 公开展示 | 每次代码提交 | ~3 min |
| P1 标准回归 | P0 + 评论 + 点赞 + 排行榜 + 标签 + 通知 | 每日构建 | ~5 min |
| P2 完整回归 | 全部 255 个测试用例 | 发布前 | ~8 min |

## 2. 回归测试集

### 2.1 P0 快速回归（核心链路）

```bash
# 后端快速回归（认证 + 作品 + 评分 + 审核 + 公开端）
mvn test -DskipTests=false -Dmaven.clean.skip=true \
  -Dtest="AuthApiTest,PublicApiTest,StudentApiTest,TeacherApiTest,AdminApiTest$Dashboard,AdminApiTest$Audit,AdminApiTest$Publish"

# 前端适配层
cd frontend && npx vitest run
```

**预期：** 全部通过（约 60+ 测试用例）

### 2.2 P1 标准回归

```bash
# 后端标准回归（全部集成测试 + 单元测试）
mvn test -DskipTests=false -Dmaven.clean.skip=true

# 前端全部测试
cd frontend && npx vitest run
```

**预期：** 255 测试全部通过，0 失败

### 2.3 P2 完整回归

在 P1 基础上增加：

```bash
# 全量测试（包含单元 + 集成 + 端到端）
mvn test -DskipTests=false -Dmaven.clean.skip=true

# 前端测试
cd frontend && npx vitest run

# 手动验证项（见 2.4）
```

### 2.4 手动回归检查项

以下场景需要手动验证（无法完全自动化）：

| 检查项 | 验证步骤 | 预期 |
|--------|---------|------|
| 文件上传大文件 | 上传 50MB+ 压缩包 | 上传成功或返回明确大小限制提示 |
| 浏览器兼容 | Chrome / Edge 打开前台作品列表和详情 | 页面正常渲染 |
| Token 过期 | 等待 token 过期后操作 | 跳转登录页 |
| DeepSeek 审核 | 提交含违规内容的作品 | 正确拦截或放行（取决于配置） |
| Redis 缓存 | 多次查询排行榜 | 缓存命中后响应速度明显提升 |

## 5. 常见回归问题

| 问题 | 影响范围 | 检测方式 |
|------|---------|---------|
| 数据库字段变更未同步到 Entity | 对应字段查询/写入异常 | P0 回归中的对应 API 测试 |
| JWT Secret 变更 | 所有已登录用户 token 失效 | REG-AUTH-01 ~ 03 |
| MyBatis-Plus 版本升级 | Mapper 方法行为变更 | 全部回归 |
| DeepSeek API 配置变更 | 内容审核结果变化 | 手工验证 |
| 前端 API 路径与后端不匹配 | 对应功能不可用 | 前端适配层测试 |

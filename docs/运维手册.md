## 1. 架构概览

```
用户 → Nginx (80)
         ├── 前端静态资源（SPA）
         └── /api/* → 后端 (8080)
后端 → MySQL (3306)
后端 → Redis (6379)
```

---

## 2. 部署方式

### 2.1 Docker 部署（推荐）

```bash
# 1. 准备环境变量
cp .env.example .env
# 编辑 .env，填写 DB_PASSWORD、DEEPSEEK_API_KEY

# 2. 一键启动
docker compose up -d

# 3. 初始化数据库
docker compose exec mysql bash -c "mysql -uroot -p\$DB_PASSWORD jingxuan < /docker-entrypoint-initdb.d/base/init_schema.sql"
# 按时间顺序执行 sql/business/ 下的增量迁移

# 4. 查看日志
docker compose logs -f

# 5. 停止
docker compose down
```

### 2.2 手动部署（PM2）

```bash
# 后端
cd /opt/jingxuan/backend
mvn package -Dmaven.test.skip=true
cp target/jingxuan-backend-1.0.0.jar .
pm2 restart jingxuan-back

# 前端
cd /opt/jingxuan/frontend
npm run build
```

## 3. 日常运维

### 3.1 进程管理

```bash
# 查看进程状态
pm2 status

# 查看日志
pm2 logs jingxuan-back           # 实时日志
pm2 logs jingxuan-back --lines 50 --nostream  # 最近 50 行

# 重启
pm2 restart jingxuan-back

# 停止
pm2 stop jingxuan-back
```

### 3.2 Nginx

```bash
# 检查配置
nginx -t

# 重载配置（不中断服务）
nginx -s reload

# 重启
systemctl restart nginx
```

### 3.3 数据库

```bash
# 连接
mysql -u root -p jingxuan

# 查看连接数
SHOW PROCESSLIST;

# 查看慢查询
SHOW VARIABLES LIKE 'slow_query%';
SHOW VARIABLES LIKE 'long_query_time';
```

### 3.4 Redis

```bash
# 连接
redis-cli

# 查看缓存状态
INFO stats

# 清空排行榜缓存
FLUSHDB

# 监控实时操作
MONITOR
```

## 4. 备份与恢复

### 4.1 数据库备份

```bash
# 全量备份（推荐每天凌晨执行）
mysqldump -u root -p jingxuan > /backup/jingxuan-$(date +%Y%m%d).sql

# 只备份结构（不含数据）
mysqldump -u root -p --no-data jingxuan > /backup/jingxuan-schema.sql

# 恢复
mysql -u root -p jingxuan < /backup/jingxuan-20260607.sql
```

### 4.2 文件备份

```bash
# 用户上传文件
tar -czf /backup/uploads-$(date +%Y%m%d).tar.gz /opt/jingxuan/uploads
```

### 4.3 定时备份（crontab）

```bash
# 编辑定时任务
crontab -e

# 每天凌晨 3 点备份数据库
0 3 * * * mysqldump -u root -p'密码' jingxuan > /backup/db/jingxuan-$(date +\%Y\%m\%d).sql
# 保留最近 30 天
0 4 * * * find /backup/db -name '*.sql' -mtime +30 -delete
```

## 5. 监控与排错

### 5.1 健康检查

```bash
# 后端状态
curl http://localhost:8080/auth/login -X POST -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
# 预期返回：{"code":200,"message":"登录成功",...}

# 数据库连接
mysqladmin ping -u root -p

# Redis 连接
redis-cli ping
# 预期返回：PONG
```

### 5.2 常见问题

| 问题 | 排查步骤 | 解决方案 |
|------|---------|---------|
| **页面 502 Bad Gateway** | `pm2 status` 检查后端是否运行 | `pm2 restart jingxuan-back` |
| **页面白屏/空白** | 浏览器 F12 → Console/Network 查看错误 | 检查前端构建是否正常 |
| **接口返回 500** | `pm2 logs jingxuan-back --lines 20` 看报错 | 根据具体异常修复 |
| **数据库连接失败** | `mysqladmin ping` 检查 MySQL | `systemctl restart mysql` |
| **上传文件失败** | 检查 `uploads/` 目录权限 | `chmod -R 755 uploads/` |
| **Redis 连接失败** | `redis-cli ping` | `systemctl restart redis` |
| **排行榜不更新** | 管理员后台手动刷新缓存 | 点击「刷新缓存」按钮 |
| **DeepSeek 审核失败** | 管理员后台 → 审核规则 → 连通性测试 | 检查 API Key 配置 |

### 5.3 日志文件位置

| 组件 | 日志位置 |
|------|---------|
| 后端（PM2） | `~/.pm2/logs/jingxuan-back-out.log` / `jingxuan-back-error.log` |
| Nginx 访问日志 | `/var/log/nginx/access.log` |
| Nginx 错误日志 | `/var/log/nginx/error.log` |
| MySQL 日志 | `/var/log/mysql/` |
| 应用日志 | Spring Boot 日志输出到 stdout（PM2 接管） |

### 5.4 JVM 监控

```bash
# 查看进程 PID
pgrep -f jingxuan-backend

# 查看内存使用
pm2 monit

# 堆转储（分析内存泄漏）
jmap -dump:live,format=b,file=/tmp/heap.hprof <PID>

# GC 日志（JVM 参数中已启用）
# 日志位置：启动目录下的 gc.log
```

## 6. 扩容与性能

| 场景 | 方案 |
|------|------|
| 用户量增加 | 增大 `-Xmx` JVM 参数（当前 2GB，服务器 7.4GB） |
| 数据库压力大 | 添加 Redis 缓存、读写分离 |
| 文件存储不足 | 挂载外部存储或迁移至 OSS |
| 单机瓶颈 | 前端 Nginx + 后端多实例部署 + 负载均衡 |

## 7. 安全注意事项

- `.env` 文件含数据库密码和 DeepSeek API Key，**严禁提交到 Git**
- 生产环境定期更换密码
- Nginx 已配置安全头（X-Content-Type-Options、CSP 等）
- 文件上传已做扩展名 + 魔数双重校验
- 默认 JWT secret 请在生产环境更换

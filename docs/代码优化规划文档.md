## 3. 优化规划

### 3.1 P0 — 安全与正确性修复（建议 1 周内）

#### P0-1：修复 SQL 注入（C-01, C-02）

**目标**：消除 `inSql()` 中所有字符串拼接

- **影响范围**：`WorkServiceImpl.java`、`WorkMemberPolicyService.java`
- **方案**：将所有 `inSql()` 调用改为安全的 `lambdaQuery` 链式调用或参数化 `IN` 查询
- **验证方式**：单元测试通过 + 手动测试作品提交/成员校验流程
- **工作量**：2 文件 × ~30 分钟 = 1 小时

#### P0-2：修复前端空 catch 块（C-04）

**目标**：消除所有静默失败

- **影响范围**：前端 20+ 个 catch 块
- **方案**：
  - 添加 `console.error` 记录错误详情
  - 移除或修正失败时显示成功消息的 catch 块
  - 添加 `ElMessage.error('操作失败：' + error.message)` 向用户反馈
- **验证方式**：检查所有 try-catch 块是否都有日志和用户提示
- **工作量**：全局搜索替换 + 逐个审查 = 2-3 小时

#### P0-3：修复硬编码 operatorId（C-03）

**目标**：奖品发放使用当前登录用户

- **影响范围**：`views/admin/prize/index.vue`
- **方案**：从 auth store 获取当前用户 ID 替代 `1`
- **工作量**：1 行改动，5 分钟

#### P0-4：Nginx 安全头（H-12）

**目标**：添加基础安全防护头

- **影响范围**：`nginx-jingxuan.conf`
- **方案**：在 server 块添加：
  - `add_header X-Content-Type-Options nosniff;`
  - `add_header X-Frame-Options SAMEORIGIN;`
  - `add_header Referrer-Policy strict-origin-when-cross-origin;`
  - `add_header Content-Security-Policy "default-src 'self' ...";`
  - `add_header Permissions-Policy "camera=(), microphone=(), geolocation=()";`
- **验证方式**：`nginx -t` 检查语法 + curl 验证响应头
- **工作量**：1 文件，30 分钟

#### P0-5：文件上传类型校验增强（C-05）

**目标**：基于文件字节真实检测 MIME 类型

- **影响范围**：`FileUploadController.java`、`FileUtil.java`
- **方案**：使用 Apache Tika 或 Hutool `FileTypeUtil.getType()` 检测真实类型，补充对扩展名和白名单的双重验证
- **工作量**：2 文件，2 小时

### 3.2 P1 — 性能优化（建议 1 月内）

#### P1-1：全表加载改为 SQL 聚合（H-01, H-02, H-03, H-04）

**目标**：消除所有将整表加载到内存的操作

- **影响范围**：
  - `AdminDashboardFacade.java` — 技术栈分布、状态分布
  - `PublicWorkFacade.java` — 公开列表查询
  - `TeacherWorkFacade.java` — 技术栈分类
  - `AdminApiController.java` — 评论 workId 列表
- **方案**：

  | 原做法 | 替换方案 |
  |---|---|
  | `selectList(null)` + Java 分组 | SQL `GROUP BY` 聚合查询 |
  | 全量 IN 子查询 + 分页 | 一次 SQL JOIN + 分页 |
  | 全表加载提取 distinct | `SELECT DISTINCT` |

- **工作量**：4-5 天

#### P1-2：消除 N+1 查询模式（H-05）

**目标**：列表查询不再逐条查关联数据

- **影响范围**：`PublicWorkFacade`、`AdminDashboardFacade`、`AdminScoreFacade`、`ScoreServiceImpl`、`WorkServiceImpl`、`AuditServiceImpl`
- **方案**：
  - 批量查询：`selectBatchIds` 或 `listByIds`
  - MyBatis-Plus `@TableName(autoResultMap=true)` + 关联映射
  - 空间换时间：一次性查全部关联数据，Java 侧 `Map<ID, T>` 索引
- **工作量**：3-4 天

#### P1-3：无界 ConcurrentHashMap 改为 Caffeine Cache（H-06, H-07）

**目标**：消除内存泄漏风险

- **影响范围**：`InMemoryTokenBlacklistService.java`、`PublicRateLimitFilter.java`
- **方案**：

  ```java
  // 替换前
  private final ConcurrentHashMap<String, Long> tokenBlacklist = new ConcurrentHashMap<>();
  
  // 替换后
  private final Cache<String, Long> tokenCache = Caffeine.newBuilder()
      .expireAfterWrite(1, TimeUnit.HOURS)
      .maximumSize(10_000)
      .build();
  ```

- **工作量**：2 文件，半天

#### P1-4：Upsert 原子化（H-09）

**目标**：消除并发场景下的重复评分

- **影响范围**：`ScoreServiceImpl.submitScore()`
- **方案**：将 select-then-insert/update 改为 `INSERT ... ON DUPLICATE KEY UPDATE`，利用数据库唯一约束保证原子性
- **工作量**：1 文件，半天
